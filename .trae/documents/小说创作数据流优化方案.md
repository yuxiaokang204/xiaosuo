# 小说创作数据流优化方案

## 问题诊断

当前系统的数据流向**流程正确**（标题→世界观→角色→大纲→章节），但**关键参数在多个环节传递不完整**，导致大模型生成的内容不能充分利用用户输入。

### 问题1：故事架构师（大纲）缺少标题
**位置**: `orchestrator.py:1015-1024` (Loop 0) 和 `orchestrator.py:1145-1156` (Loop 1)

**现状**:
```python
# Loop 0 S5 大纲骨架
outline_result = await agents["story_architect"].process({
    "theme": self.state.theme,
    "tone": self.state.tone,
    "chapter_count": self.state.chapter_count,
    "platform": self.state.platform,
    "world_info": self._stringify(self.state.world_settings)[:400],  # ← 只传400字符
    "characters": self._stringify([c.get("name", "") for c in self.state.characters]),  # ← 只传名字
    "depth_level": 0,
    # ❌ 缺少 title
    # ❌ 缺少 story_direction
})
```

**影响**: 大纲设计不知道小说标题，无法根据标题主题设计章节结构。

### 问题2：世界观截断过短
**现状**: `world_info[:400]` 只传递前400字符，世界观JSON通常包含规则、地点、势力、历史等，400字符严重截断。

### 问题3：story_direction 生成时机
**现状**: story_direction 在 Loop 0 S5（大纲骨架）之后生成，但 Loop 1 逐章写作时需要引用它。

### 问题4：角色只传名字不传设定
**现状**: `characters` 只传 `[c.get("name", "") for c in self.state.characters]`（名字列表），大纲师不知道角色性格、目标等。

## 优化方案

### 方案总览

```
用户输入: title + theme + tone + chapter_count
              │
              ▼
┌─────────────────────────────────────────┐
│  Loop 0: SKELETON 骨架层                 │
│  ┌─────────────┐    ┌──────────────┐    │
│  │ S1 世界观    │────│ S2 角色       │    │ ← 标题传入
│  │ title +     │    │ title +      │    │    世界观传入
│  │ theme +     │    │ world_info   │    │
│  │ tone +      │    │ + theme +    │    │
│  │ tone        │    │ tone         │    │
│  └─────────────┘    └──────────────┘    │
│         │               ↑                │
│         ▼               │                │
│  ┌──────────────────────────┐            │
│  │ S5 大纲骨架              │            │
│  │ title +                  │            │
│  │ full world_settings      │            │
│  │ full characters (含设定)  │            │
│  │ chapter_count            │            │
│  └──────────────────────────┘            │
│         │                                 │
│         ▼                                 │
│  ┌─────────────┐                         │
│  │ S6 故事走向  │ ← 基于 title + outline  │
│  └─────────────┘                         │
└─────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────┐
│  Loop 1: DETAIL 细化层                   │
│  ┌─────────────┐    ┌──────────────┐    │
│  │ S1 世界观细化│────│ S2 角色细化   │    │
│  │ + title     │    │ + title +    │    │
│  │ + 已有设定   │    │ 已有设定      │    │
│  └─────────────┘    └──────────────┘    │
│         │               ↑                │
│         ▼               │                │
│  ┌──────────────────────────┐            │
│  │ S5 大纲细化              │            │
│  │ title +                  │            │
│  │ full world + full chars  │            │
│  │ + existing outline       │            │
│  │ + story_direction        │ ← v6.0     │
│  └──────────────────────────┘            │
│         │                                 │
│         ▼                                 │
│  ┌──────────────────────────┐            │
│  │ 逐章写作（所有章节）      │            │
│  │ 每章接收:                 │            │
│  │ - title + theme + tone   │            │
│  │ - full world_settings    │            │
│  │ - full characters        │            │
│  │ - outline + story_dir    │            │
│  │ - roleplay_cards (v6.0)  │            │
│  └──────────────────────────┘            │
└─────────────────────────────────────────┘
```

#### 4. build_story_architect_user_prompt 支持 title + story_direction

**文件**: `src/backend/agents/prompts.py`
**改动**: 函数签名新增 `title` 和 `story_direction` 参数

```python
def build_story_architect_user_prompt(
    title: str = "",
    theme: str = "",
    tone: str = "",
    chapter_count: int = 10,
    platform: str = "番茄",
    world_info: str = "",
    characters: str = "",
    start_chapter: int = 0,
    existing_outline: str = "",
    story_direction: str = "",  # 新增
    depth_level: int = 1,
) -> str:
    parts = []
    if title:
        parts.append(f"小说标题：《{title}》")
    parts.append(f"故事类型：{theme}")
    parts.append(f"风格基调：{tone}")
    # ... 其余逻辑
    if story_direction:
        parts.append(f"\n【故事走向指引】\n{story_direction}")
    return "\n".join(parts)
```

## 总结

| 环节 | 当前问题 | 修复方案 |
|------|---------|---------|
| Loop 0 S5 大纲 | 缺 title、world[:400]、chars[:400] | 传 title + 完整 world + 完整 chars |
| Loop 1 S5 大纲 | 缺 title、story_direction | 传 title + story_direction + 完整 world/chars |
| story_architect_agent.py | 未处理 title + story_direction | 新增参数提取和提示词追加 |
| build_story_architect_user_prompt | 无 title + story_direction 参数 | 新增参数 |

**核心原则**: **标题是灵魂、世界观是土壤、角色是种子、大纲是蓝图**。所有阶段的 LLM 调用都应尽可能拿到完整的前置数据，不做不必要的截断。
