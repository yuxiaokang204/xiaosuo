# 小说标题数据流断裂修复方案

## 问题诊断

标题"直播研发丧尸病毒，全网吓哭"在**run_stage（线性模式）**的以下关键阶段丢失：

| 阶段 | title传递状态 | 影响程度 |
|------|--------------|---------|
| worldbuilding | ✅ 完整传递 | 无影响 |
| characters | ✅ 完整传递 | 无影响 |
| **opening_hook** | ❌ 未传递 | **严重** - 黄金三章设计与标题无关 |
| **outlining** | ❌ 未传递 | **严重** - 大纲设计与标题无关 |
| style | ⚠️ 未传递 | 次要影响 |

**根本原因**: `orchestrator.py:run_stage` 的 `opening_hook` 和 `outlining` 阶段在调用 `agents["xxx"].process()` 时，没有传递 `"title"` 参数。虽然 title 存储在 `self.state.title` 中，但 Agent 的 `execute()` 方法通过 `context.get("title", "")` 获取，拿到的是空字符串。

## 修复方案

### 1. 修复 opening_hook 阶段 — 新增 title 传递

**文件**: `src/backend/core/orchestrator.py`
**位置**: 约549-564行

**当前代码**:
```python

    world_json = json.dumps(self.state.world_settings, ensure_ascii=False) if self.state.world_settings else ""
    char_json = json.dumps(self.state.characters, ensure_ascii=False) if self.state.characters else ""
    result = await agents["opening_hook"].process({
        "theme": self.state.theme,

```

**修改为**:
```python

    world_json = json.dumps(self.state.world_settings, ensure_ascii=False) if self.state.world_settings else ""
    char_json = json.dumps(self.state.characters, ensure_ascii=False) if self.state.characters else ""

        "title": self.state.title,
```
